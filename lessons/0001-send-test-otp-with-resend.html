ส่ง OTP ทดลองด้วย Resend

เป้าหมาย: ให้ Edge Function ส่งจาก Dilion <onboarding@resend.dev> เข้าอีเมลของคุณ โดยไม่ทำ API Key รั่ว

ขั้นที่ 1 — สร้างบัญชี
  1. เปิด Resend Sign Up และสมัครด้วยอีเมลที่คุณจะใช้รับ OTP
  2. ยืนยันอีเมลและเข้าสู่ Dashboard
  3. จำไว้ว่าโหมดทดลองส่งเข้าอีเมลบัญชีนี้เท่านั้น
ขั้นที่ 2 — สร้าง API Key
  1. เข้าเมนู API Keys แล้วกด Create API Key
  2. ตั้งชื่อ aiot-school-lab-local
  3. เลือกสิทธิ์ Sending access ไม่ต้องเลือก Full access
  4. คัดลอกคีย์ที่ขึ้นต้นด้วย re_ เก็บไว้ชั่วคราว ระบบอาจไม่แสดงซ้ำ

อ้างอิง: Resend API Keys

ขั้นที่ 3 — ใส่ Secret ในโปรเจกต์

ที่โฟลเดอร์ /Users/sayfa/my_first_app คัดลอกไฟล์ตัวอย่าง:

cp supabase/functions/.env.example supabase/functions/.env

เปิด supabase/functions/.env แล้วแทนค่าเฉพาะบรรทัดแรก:

RESEND_API_KEY=re_คีย์จริงของคุณ
RESEND_FROM_EMAIL=Dilion <onboarding@resend.dev>

Supabase จะโหลดไฟล์นี้อัตโนมัติตอน supabase start ตาม Supabase Secrets Guide

ขั้นที่ 4 — โหลด Secret และทดลอง

ถ้า Supabase ทำงานอยู่ ให้หยุดแล้วเริ่มใหม่เพื่อโหลด Secret:

npx supabase stop
npx supabase start

จากนั้นใช้หน้าขอ OTP/รีเซ็ตรหัสผ่านของแอป โดยกรอกอีเมลเดียวกับบัญชี Resend แล้วตรวจ Inbox และ Spam

กติกาความปลอดภัย
  • อย่าส่ง API Key มาในแชต แม้ต้องการให้ช่วยตั้งค่า
  • อย่าใส่คีย์ไว้ใน Flutter เพราะผู้ใช้สามารถดึงคีย์จากแอปได้
  • ถ้าคีย์เคยหลุด ให้ลบทิ้งใน Resend และสร้างใหม่ทันที

เช็กว่าทำสำเร็จ

  • Resend Dashboard แสดงสถานะอีเมลว่า Delivered
  • อีเมลเข้าบัญชีเดียวกับที่สมัคร Resend
  • ชื่อผู้ส่งแสดงเป็น Dilion
  • OTP ใช้งานได้และหมดอายุตามที่ระบบกำหนด
