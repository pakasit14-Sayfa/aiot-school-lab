## 3. งานที่ทำโดย Codex

### 3.1 เตรียมเครื่องและฐานข้อมูลทดสอบ

- กู้ Docker Desktop จากสถานะค้างหลังดิสก์เคยเต็ม
- เริ่ม Supabase แบบเลือกเฉพาะบริการที่จำเป็น เพื่อลดการใช้ดิสก์
- เฝ้าพื้นที่และหยุดการดาวน์โหลดชุดเต็มก่อนดิสก์เต็มซ้ำ
- Apply migrations ทั้งหมดจากฐานข้อมูลว่างสำเร็จ

### 3.2 Session Token

- เพิ่ม `SessionTokenStorage`
- ย้าย `session_token` จาก `SharedPreferences` ไป `flutter_secure_storage`
- ย้าย token เก่าเข้าพื้นที่ปลอดภัยหนึ่งครั้งและลบ plaintext copy
- เก็บ token ในฐานข้อมูลเป็น SHA-256 hash เท่านั้น
- จำกัดไม่เกิน 5 active sessions ต่อบัญชี
- เพิ่ม `auth_sign_out_all` สำหรับออกจากระบบทุกอุปกรณ์
- เชื่อมปุ่ม “ออกจากระบบทั้งหมด” กับ RPC จริง
- ลบไฟล์ auth จำลองที่เก็บอีเมลและรหัสผ่าน plaintext ในเครื่อง

### 3.3 Login Rate Limit

- เพิ่มตารางจำกัดความถี่ที่ไม่เปิดให้ client อ่านโดยตรง
- จำกัดตามบัญชี: ผิด 5 ครั้งใน 15 นาที บล็อก 15 นาที
- จำกัดตาม IP: ผิด 20 ครั้งใน 15 นาที บล็อก 15 นาที
- สร้าง HMAC-SHA256 IP fingerprint ด้วย server-side pepper ก่อนส่งเข้าฐานข้อมูล
- ล้างค่า IP plaintext จาก session flow เก่า และเก็บเฉพาะ fingerprint 64 ตัวอักษร
- สร้าง Edge Function `auth-sign-in`
- ถอนสิทธิ์ `anon` และ `authenticated` ไม่ให้เรียก RPC ล็อกอินตรง
- ให้ Edge Function ใช้ `service_role` เรียก RPC ภายใน
- ทำผลตอบกลับแบบกลาง เพื่อลดการเดาว่าบัญชีมีอยู่จริงหรือไม่

### 3.4 Parent Binding และ Consent

- Binding Code อายุ 7 วัน ใช้ครั้งเดียว และจำกัดไม่เกิน 2 ผู้ปกครองต่อนักเรียน
- เพิ่ม OTP ทางอีเมล อายุ 10 นาที และจำกัดการกรอกผิด
- ใช้ opaque verification token ไม่คืนข้อมูลนักเรียนให้ public request
- ยังไม่ให้ Parent Role ก่อนโรงเรียนอนุมัติ
- บล็อกการอนุมัติคำขอของตนเอง
- เพิ่ม second review โดย School Admin คนละคน
- เพิ่มการปฏิเสธพร้อมเหตุผลและ Audit Log
- เพิ่มหน้าผู้ปกครองสำหรับดู ให้ และถอน Consent
- เก็บ Consent ตาม Policy Version, Document URL และ SHA-256
- เก็บ `consent_events` แบบ append-only
- เพิ่มหน้า School Admin สำหรับ publish/retire Consent Policy
- บังคับ Policy URL เป็น HTTPS และ Document Hash เป็น SHA-256 64 ตัวอักษร
- แยก Policy เป็นแบบจำเป็นและกิจกรรมเสริม
- การ retire ไม่ลบ Policy หรือหลักฐาน Consent เดิม

### 3.5 RLS/RPC และการแยกโรงเรียน

- เปิด RLS ครบทุกตารางใน `public`
- ถอน direct table/sequence privileges จาก `PUBLIC`, `anon` และ `authenticated`
- ถอน implicit `PUBLIC EXECUTE` จาก `SECURITY DEFINER` functions
- ใช้ RPC-only ตาม custom session ของระบบ
- ตรวจ Active Role และ Active School ใน RPC
- เพิ่ม negative tests ป้องกันอ่าน/แก้ผู้ใช้และให้ Role ข้ามโรงเรียน

หมายเหตุ: ระบบนี้ไม่ได้ใช้ Supabase Auth JWT จึงไม่ใช้ direct-table policy แบบ `auth.uid()` การป้องกันที่ทำงานจริงคือ deny-all RLS + ถอน table grants + allow-listed RPC + cross-school tests

### 3.6 งานตรวจและแก้เพิ่มเติม

- แก้ SQL ที่ชื่อ output parameter `status` ชนกับชื่อ column
- เพิ่มหน้าและ service สำหรับ Consent Policy lifecycle
- ลบข้อมูลอุปกรณ์จำลองในหน้า Profile ที่เคยแสดงชื่ออุปกรณ์และเวลาปลอม
- ตรวจ source ไม่พบ API key, session token หรือรหัสผ่านจริงถูกเขียนค้างไว้
- สร้างรายงานรายละเอียดที่ `docs/pdpa-security-implementation-2026-07-22.md`

### 3.7 แก้ผลตรวจจาก Claude `/redteam`

- Claude พบว่า `auth_sign_in` ข้าม `crypt()` เมื่อไม่พบ user ทำให้วัดเวลาเดาบัญชีได้
- Codex สร้าง timing reproduction พบ 193.8 ms สำหรับบัญชีที่มี เทียบ 2.4 ms สำหรับบัญชีที่ไม่มี
- Codex เพิ่ม dummy bcrypt hash และบังคับให้ทุก credential check เรียก `crypt()`
- หลังแก้วัดได้ 92.5/80.0 ms และ regression test ผ่าน
- Claude พบว่า Parent OTP รอ Resend เฉพาะกรณี Binding Code ตรง
- Codex ย้าย email delivery ไป background task ด้วย `EdgeRuntime.waitUntil`
- Codex เพิ่ม minimum response time 350 ms ทั้ง valid และ invalid path
- HTTP warm timing หลังแก้: invalid 0.366 วินาที และ valid 0.375 วินาที
- Codex เพิ่ม `operational_alerts` และ optional `SECURITY_ALERT_WEBHOOK_URL`
- ทดสอบ provider config หายแล้ว durable alert ถูกสร้างจริง
- Codex เปลี่ยน IP เป็น HMAC-SHA256 fingerprint ด้วย `LOGIN_IP_PEPPER`
- ทดสอบ session เก็บค่า 64 ตัวอักษรและไม่มีรูปแบบ IPv4 plaintext

## 6. งานที่ยังไม่เสร็จ

### ต้องทำต่อก่อน Deploy

- Deploy migration และ Edge Functions ไป Supabase remote แบบพร้อมกัน
- ตั้งค่าและทดสอบผู้ส่งอีเมล OTP จริง

### ต้องทำก่อนใช้ข้อมูลส่วนบุคคลจริง

- ให้โรงเรียน/DPO/ที่ปรึกษากฎหมายกรอกและอนุมัติ Privacy Notice
- ยืนยัน Controller/Processor และฐานกฎหมายแต่ละกิจกรรม
- อนุมัติ Retention, DPA และการโอนข้อมูลไปต่างประเทศ
- ทำ 2FA, Trusted Device และ Active Role/Active School ตาม scope F02 ที่อยู่นอกงาน 4 gate รอบนี้
- ห้ามเปิดกล้องหรือข้อมูลชีวมิติจนกว่า Camera DPIA จะผ่าน

### ต้องใช้คนหรือฮาร์ดแวร์จริง

- ทดสอบ Relay กับบอร์ด ไฟ หรือปั๊มน้ำจริง
- ยืนยัน Sensor อุณหภูมิ/ความชื้นหลังติดตั้ง Sensor จริง
